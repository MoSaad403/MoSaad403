## About

Modern software isn't defined only by the applications users interact with—it's shaped by the infrastructure that powers them.

I enjoy building resilient cloud platforms, automating operational workflows, and designing infrastructure that is reproducible, observable, and easy to maintain. My interests span cloud architecture, Kubernetes, Infrastructure as Code, CI/CD, Linux, and platform engineering, with a strong appreciation for open-source technologies that make engineering teams faster and more reliable.

I believe the best infrastructure is the one developers rarely have to think about.

<summary><b>Engineering Philosophy</b></summary>

<br>

> Infrastructure should be predictable before it is powerful.

I value systems that are:

- Automated rather than repetitive.
- Observable rather than assumed.
- Reproducible rather than manually configured.
- Reliable before feature-rich.
- Simple enough to maintain and scalable enough to grow.

Good engineering isn't measured by complexity—it's measured by clarity, resilience, and the confidence to deploy repeatedly.
